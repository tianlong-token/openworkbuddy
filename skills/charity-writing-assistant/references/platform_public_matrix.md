# 六大互联网公募平台项目上线字段速查

> 基于《六大互联网公募平台项目上线填写内容对照表》提炼，用于 Step A1 平台选择和 Step A2 字段预判。

## 平台覆盖分类（首次引导优先用这套说法）

- **互联网公募平台**：腾讯公益、字节跳动公益、支付宝/蚂蚁公益、微博微公益、京东公益、美团公益
- **垂直筹款平台**：轻松筹、水滴筹、联劝网等
- **基金会/枢纽平台**：恩派公益基金会、中国慈善联合会及各地方慈善会平台
- **其他/未列出平台**：按通用公益规范 + 该平台特点生成

## 六大平台快速判断

| 平台 | 更适合的项目 | 上线时必须先问清的字段 | 关键差异 |
|------|-------------|----------------------|---------|
| 腾讯公益 | 标准公募筹款项目 | 9字项目名、27字简介、背景/故事/方案、预算、执行能力、配图 | 字数限制最严 |
| 字节跳动公益 | 需要短视频传播的项目 | 项目视频、传播计划、捐赠人服务计划、肖像授权 | 视频几乎刚需 |
| 支付宝/蚂蚁公益 | 逻辑完整、执行规范的项目 | 背景→目标→计划→预算→执行→票据方式完整链路 | 评审严格 |
| 微博微公益 | 有KOL/转发传播资源的项目 | 是转发捐助还是长期项目、转发规则、原文内容、海报 | 多个字段发布后不可改 |
| 京东公益 | 资金募捐或物资募捐项目 | 先分资金/物资类型、物资SKU清单、接收地址、筹款天数 | 物资募捐是特色 |
| 美团公益 | 本地社区类项目 | 项目领域、受益人数、预算、执行团队、风险控制 | 本地生活场景更强 |

## 细分场景直达

- 用户说“微博转发捐助” → 直接走微博转发捐助字段，不要按普通项目上线处理
- 用户说“京东物资募捐” → 直接采集物资清单、接收地址、截止方式
- 用户说“字节/抖音公益” → 先确认视频素材是否具备
- 用户说“支付宝公益” → 先检查完整链路是否齐全，尤其票据获取方式
- 用户说“美团公益” → 先问项目是否具备社区/本地生活场景优势

## 通用准备材料清单

- 机构登记证书/营业执照扫描件
- 公募资质证明或公募合作协议
- 项目名称（准备多个长度版本）
- 项目简介（短版 / 中版 / 完整版）
- 项目背景文案（含政策/行业数据）
- 受益对象故事（至少 1-2 个，已获授权）
- 详细预算表
- 执行计划和进度安排
- 执行机构/团队介绍
- 项目封面图与详情配图
- 肖像授权书（如使用人物照片/视频）

> 规则可能更新；若联网可用，仍需再用 web_search 对照平台最新要求。
