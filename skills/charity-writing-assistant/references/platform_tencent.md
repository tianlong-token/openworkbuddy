# 腾讯公益 — 平台格式规范

## 项目上线申请

**字数限制（硬性）**：项目名称 ≤ **9字** | 项目简介/一句话描述 ≤ **27字** | 副标题 ≤ **40字** | 背景详情 200-1000字 | 受益对象故事 每篇 ≤ 1000字 | 帮扶方案 200-1000字

### 必填模块

1. **基础信息**：项目名称(≤9字)、一句话描述(≤27字)、副标题(≤40字，可选)、筹款目标金额、执行周期/筹款天数、受益人数、目标地区
2. **项目背景**：背景详情需写清问题现状、政策/行业数据、当地需求，建议采用“问题现状 → 为什么现在做 → 为什么由你们来做”的顺序
3. **受益对象故事**：至少 1 篇，建议 2-3 篇。可写困境故事或改变故事，但必须保护受助人尊严与隐私，不得捏造或夸大
4. **帮扶方案/执行计划**：明确做什么、怎么做、谁来做、何时做，最好写出活动频次、服务方式和量化目标
5. **预算明细表**：逐项列明费用用途、单价、数量、金额、备注；管理费通常不超过总筹款的 10%
6. **执行能力**：执行机构介绍、执行团队信息、过往经验、项目地资源基础
7. **呼吁与反馈**：呼吁倡导文案、捐赠回馈方案（可选，但有利于审核与传播）
8. **图片/物料**：项目封面图、详情页配图（≥3张）、机构 logo/证件

### 关键注意事项

- **公募资格**：个人和非公募机构发起项目，需由具备公募资格的慈善组织认领审核
- **审核偏好**："小而美"、预算清晰、故事真实、有反馈机制的项目更容易通过
- **预算规范**：管理费（行政成本）一般不超过总筹款的 10%
- **传播节点**：99公益日/久久公益节是流量高峰，建议提前 2-3 个月准备上线材料

## 中期/结项报告

**必填字段**：项目名称、受助地区、简介、筹款目标/实际筹款、公募机构/执行机构、公募管理费、实际总支出、直接/间接受益人群

**善款使用明细表**（必须）：序号 | 支出事项 | 数量 | 单价 | 预算小计 | 实际支出 | 备注。末行“执行机构管理费”标明占比，最后一行“支出合计”。⚠️ 支出合计 = 实际总支出 + 公募管理费，数据必须一致

**其他必备**：票据清单（时间须在项目上线之后）、执行现场图片（**≥5张**）、执行过程详述、项目成果（量化）

中期报告为阶段性数据，结项报告需完整最终数据。

## 📌 平台操作指引

生成的文书内容已按腾讯公益乐捐平台后台表单字段顺序排列，可直接复制对应段落到相应输入框。

- **平台入口**：https://open.gongyi.qq.com
- **常用路径**：登录后进入「发起项目」或已有项目后台 → 填写/更新项目信息 → 提交审核
- **操作建议**：逐段复制粘贴，重点复核 9 字项目名、27 字简介和预算表；超限内容平台会自动截断
- **图片上传**：项目图片和执行照片需在平台后台单独上传，文书中的图片描述可作为 ALT 文本
