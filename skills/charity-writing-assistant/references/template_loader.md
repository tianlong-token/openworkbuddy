# 模板使用说明 — 执行步骤

确定用户选择的平台后，按以下三层机制保障模板内容的可用性、时效性和准确性。

## 索引表地址

`https://docs.qq.com/sheet/DQm9tR2FqZ2Znd2FU?tab=BB08J2`

## 第 1 层：预置模板

本地 `references/` 预置基线模板，确保断网、Playwright 不可用、远端故障等任何异常下 Skill 都能正常生成文书。

## 第 2 层：在线热更新

当环境支持 `playwright-cli` 时执行：

1. **检查是否需要更新**：比对本地 `references/platform_{类型}.md` 文件修改时间，距今超过 **7 天** 或文件不存在 → 触发更新
2. **拉取索引表**：
   ```
   playwright-cli open {索引表地址}
   → waitForLoadState('networkidle') + 0.5s 缓冲
   → grantPermissions(['clipboard-read','clipboard-write'])
   → click 激活 → Meta+a → Meta+c → clipboard.readText()
   ```
3. **解析索引**：按 `\t` 分列解析每行的 `文件名、类型(platform_xxx)、更新时间`
4. **比对更新时间**：远端更新时间 > 本地文件修改时间 → 需要更新该模板
5. **拉取子文档**：点击索引表中对应行的超链接（自动打开新 tab）→ 切到新 tab → 同样 networkidle + 全选复制 → 拿到正文
6. **写入本地**：将正文写入 `references/platform_{类型}.md`，覆盖旧版
7. **关闭浏览器**

**降级规则**：
- Playwright 不可用 → 跳过热更新，用本地预置版本
- 索引表打开失败 → 跳过热更新，用本地预置版本
- 子文档抓取失败 → 该模板用本地版本，不影响其他模板
- 任何失败都不报错、不中断流程

## 第 3 层：实时字段校验

即使模板已是最新版，仍在正式向用户提问前执行一次实时校验：

1. 用 `web_search` 搜索 `"{平台名} 项目上线 填写字段 最新要求 {当前年份}"`
2. 将搜索结果与当前模板 + `collection_fields.json` 字段清单快速比对
3. 发现新增/变更 → 临时追加采集清单 + 提示用户确认
4. 离线降级 → 跳过校验，用本地字段，文书末尾注明建议核对

## 加载优先级

```
用户选了平台
  ↓
1. 本地有 platform_{平台}.md？
   ├── 有 + 未过期(≤7天) → 直接用，走第3层校验
   ├── 有 + 已过期(>7天) → 尝试第2层热更新
   │     ├── 成功 → 用新版，走第3层校验
   │     └── 失败 → 用旧版，走第3层校验
   └── 无 → 尝试第2层热更新
         ├── 成功 → 用新版，走第3层校验
         └── 失败 → 用 platform_common.md + collection_fields.json 兜底
  ↓
第3层校验完成 → 开始向用户分步采集
```
